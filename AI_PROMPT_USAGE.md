## 2. 주제 · 기술 스택 확정

**프롬프트**
> (로또 생성기 3-tier 표 이미지(클라우드.txt파일) 첨부) "이런 식으로 진행될 거야.
> DB는 MySQL, Web엔진은 nginx, Application은 Python Flask를 사용할 거야."

**용도**
- 만들 서비스의 **주제(로또 번호 자동 생성기)** 와 **기술 스택**을 확정
  - Presentation: nginx
  - Application: Flask
  - Data: MySQL

**결과 반영**
- 확정된 스택대로 폴더 구조와 각 컨테이너를 구성
  (`presentation/`=nginx, `application/`=Flask, `data/`=MySQL)
- 표에 적힌 역할(요청 수신 / 번호 6개 추첨·DB 처리 / 결과 저장·조회)을
  그대로 기능으로 구현

## 3. 전체 프로젝트 구현 요청

**프롬프트**
> "위 스택대로 3-tier 로또 생성기를 만들어 줘. docker-compose로 한 번에 실행되게."

**용도**
- 실제 소스코드 일체 생성 (Dockerfile, nginx 설정, Flask 앱, init.sql, docker-compose, 프론트엔드 화면)

**결과 반영 / 검토한 점**
- AI가 생성한 코드를 받아 **그대로 채택하되 아래를 확인**함
  - `docker-compose.yml` : 3개 서비스가 `lotto-net` 네트워크로 연결되고
    `depends_on` + `healthcheck`로 DB 준비 후 앱이 뜨도록 구성됨을 확인
  - Flask가 DB 접속 정보를 **환경변수**로 받도록 되어 있어, compose 설정과 일치하는지 확인
  - 추첨 로직(`random.sample`)이 1~45·중복 없이 6개·정렬 조건을 만족하는지 테스트로 검증
  - nginx의 `/api/` 프록시 대상이 `app:5000`(서비스 이름)으로 맞는지 확인
- 프론트엔드는 **공식 로또 공 색상**(구간별 색)을 적용하자는 AI 제안을 수용

## 4. 실행 오류 디버깅 (DB 연결 500 에러)

**프롬프트**
> (Docker 로그 첨부) "컨테이너 3개 열리는 거 확인하고 localhost도 잘 열리는데,
> 로또 번호 추첨이 안 돼."

**용도**
- 번호 생성 시 발생한 `500 Internal Server Error`의 원인 파악 및 수정

**결과 반영**
- 로그의 핵심 메시지 확인:
  `RuntimeError: 'cryptography' package is required for ... caching_sha2_password`
- 원인: **MySQL 8.0의 기본 인증 방식(`caching_sha2_password`)** 을 PyMySQL이
  처리하려면 `cryptography` 패키지가 필요한데 누락되어 있었음
- 해결: `application/requirements.txt`에 `cryptography==42.0.8` 추가 후
  `docker compose up --build`로 app 컨테이너 재빌드 → 정상 동작 확인

## 작성 팁 (감점 방지)

과제 안내에 "사용 내역을 누락하거나 너무 단순한 프롬프트 구성 시 감점될 수 있음"이라고 명시되어 있습니다.

- 프롬프트를 **요약하지 말고 실제로 입력한 문장**을 적기
- 코드 한 줄을 고치더라도 "왜 요청했고(용도) 어떻게 반영했는지(결과)"를 함께 적기
- AI가 제안한 것을 **그대로 쓴 부분 / 직접 수정한 부분**을 구분해서 적으면 신뢰도가 올라감
